HAL System (ATmega32A)

Master initialization, self-test, and emergency shutdown orchestration.

Master initialization, self-test, and emergency shutdown orchestration.

Overview

  • Central entry points for bringing up all hardware subsystems in safe order.
  • Provides unified self-test routine and emergency shutdown capability.

Init Order & Rationale

1) timer_init() ␐→ provides timing for subsequent subsystems 2) gpio_init() ␐→ ensures outputs low, inputs pull-up 3) adc_init() ␐→ analog ready before PWM enables any drive 4) pwm_init() ␐→ amplitude path configured but disabled by default 5) uart_init() ␐→ MODBUS/Serial available last to report status

Safety

  • On failure of any stage, initialization stops and maps to HAL error code.
  • Emergency shutdown disables PWM, drives START low, clears UART, and kicks watchdog.
